## 산정기준 및 해석 원칙
- 공식 이수 판정 KPI는 `본(재)평가` 최종점수만 사용합니다.
- 본평가 후 재평가를 실시한 훈련생은 최종 재평가 점수를 기준으로 적용합니다.
- 공식 이수 판정은 `60점 이상 PASS / 60점 미만 FAIL`입니다.
- `사전평가`, `사전진단`, `SELF CHECK`, `프로젝트 참여도`는 공식 이수 판정 점수에 별도로 산입하지 않습니다.
- 사전평가와 사전진단은 본(재)평가와 비교해 개인별 성장세와 학습 전후 변화를 해석하는 자료입니다.
- SELF CHECK 이론 점검은 기초 이론 누락 여부를 확인하는 자료입니다. 점수가 높다면 수업 중 이론 전달이 안정적이었고 훈련생이 기초 이론을 놓치지 않았다는 근거로 해석합니다.
- 능력단위7은 `능력단위7 PBL`로 표기하고, 정식 기업맞춤 프로젝트와 분리합니다.
- `능력단위7 PBL`과 `프로젝트1~3`에는 각 평가기준 안에 협업 요소가 포함되어 있으므로 별도 `프로젝트 참여도 25%`를 공식 평가 기준으로 두지 않습니다.
- 다만 기업 제안용 협업 지표가 필요할 때는 프로젝트별 협업 점수로 `동료평가:자기평가:교수자 프로젝트 평가의견서 협업 부문 = 3:2:5` 비율을 적용합니다.
- 교수자 평가의견서에 서술된 협업 관련 정성 의견은 점수 산식에 기계적으로 더하지 않고, 강점·보완점·기업 전달 의견 작성 시 참작합니다.
- 채용 적합도는 공식 이수 판정이 아니라 기업 제안용 내부 종합 지표입니다. 현행 비율 `본(재)평가 40% + SELF CHECK 20% + 프로젝트 협업/수행 지표 25% + 출석률 15%`는 유지하되, PASS/FAIL 판정과 분리해서 사용합니다.

## 프로젝트 구분 원칙
- 능력단위7 팀플은 정식 기업맞춤 프로젝트가 아니라 `능력단위7 PBL`로 표기합니다.
- 정식 기업맞춤 프로젝트는 `프로젝트1~프로젝트3`입니다.
- 매칭 기준: `프로젝트1=기업1`, `프로젝트2=기업2`, `프로젝트3=기업3`.
- 추후 기맞최종 현장평가에서는 기업별 대시보드/보고서 3벌을 유지하고, 개인 최종 1장은 `1~6능력단위 총 성적 + 능력단위7 PBL 1개 + 프로젝트 3개` 구조로 확장합니다.
