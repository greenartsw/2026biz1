# 개인별 성과 리포트·Apps Script 통합 디버깅 기록

- 작업일: 2026-07-15
- 대상 저장소: `greenartsw/2026biz1`
- 대상 기능: 개인별 성과 리포트, 기업 피드백 복원, 팀 표지 조회, 학생 토큰 링크, Apps Script 발송 관리
- 보안: 학생 실명, 이메일, 접근 토큰은 이 기록에서 제외함

## 최초 현상

- 전날 Apps Script에서 테스트 메일과 학생 메일 발송까지 정상 완료했다.
- 다음 날 학생이 기존 메일 링크로 다시 접속하자 개인 성적은 표시되지만 기업 피드백은 빈 입력폼으로 나타났다.
- 해당 학생의 선정 팀 표지 화면은 정상 표시됐다.
- 처음에는 토큰 인식 실패가 의심됐다.

## 확인 결과

- 기존 학생 토큰은 Apps Script에서 정상 인식됐다.
- 토큰 링크는 올바른 학생의 GitHub Pages 리포트 iframe을 생성했다.
- 개인 성적 데이터와 팀 표지 데이터도 정상 제공됐다.
- `action=completed` 응답만 완료 학생과 기업 피드백 record를 빈 배열로 반환했다.

## 근본 원인

Google Sheets의 평가 헤더에는 화면 정렬을 위한 줄바꿈이 포함돼 있었다.

예시:

- 코드 기준: `콘텐츠 기획 및 구성의 적절성`
- 실제 시트: `콘텐츠 기획 및\n구성의 적절성`

기존 Apps Script는 헤더 문자열을 완전히 일치시켰다. 따라서 저장된 평점 다섯 개를 찾지 못했고, 완료 피드백 record로 인정하지 않았다. 전날 저장 직후 화면은 브라우저가 방금 처리한 상태를 가지고 있어 정상처럼 보일 수 있었지만, 다음 날 새로 접속한 화면은 `action=completed`를 다시 호출하면서 문제를 드러냈다.

## 적용한 수정

- 헤더 비교 시 공백과 줄바꿈을 제거한 정규화 키를 사용한다.
- 줄바꿈이 포함된 기존 헤더와 줄바꿈 없는 코드 헤더를 같은 항목으로 처리한다.
- 완료 상태 조회에서도 정규화된 헤더로 다섯 개 평점을 복원한다.
- 새 피드백 저장 시에도 기존 줄바꿈 헤더에 정확히 값을 기록한다.
- 같은 의미의 헤더를 뒤쪽 열에 중복 추가하지 않는다.
- 잘못된 JSON이나 필수 정보가 없는 POST 요청은 빈 행으로 저장하지 않는다.
- 팀1·팀2·팀3이 모두 저장돼야 표지 선정 완료로 판단한다.
- 학생 토큰에서 다른 팀의 표지를 임의로 요청하지 못하게 제한한다.
- 발송 동시 실행에는 Apps Script 전체 범위 잠금을 사용한다.
